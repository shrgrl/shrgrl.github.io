What is Github Pages?

First, let's briefly talk about the features of Github Pages. GitHub Pages is a static hosting service that takes HTML, CSS and JavaScript files directly from a repository on GitHub and publishes a website. You can get more detailed information from here and examine Github's own example collection. The first and most important step in creating a Github Pages site is the naming standard. As you will see in the example we will do soon, the rule is:

{% highlight javascript %} your-uname.github.io {% endhighlight %}

And thus we have created an empty repository:


Now, when we look into the empty repository to push the repository we examined earlier to our own repository, it offers us some options. Now we will clone the project we examined with Git Bash and push it to our own repository.


Here, before starting the push process, we need to clone the repository we based on:

{% highlight javascript %} git clone https://github.com/your-uname/reponame.git {% endhighlight %}

After cloning, let's navigate to this file's directory:

{% highlight javascript %} cd reponame {% endhighlight %}

And let's perform the following operations respectively:

{% highlight javascript %} git init git remote set-url origin https://github.com/your-uname/your-uname.github.io.git git branch -M main git push -u origin main {% endhighlight %}

Finally, we need to have pushed without getting such an error.


After this process, when we check our repository content, we see that we have copied it without any problems.


Now you can run this project with the your-uname.github.io URL.
