Hello everyone. Today I'm going to show you some information about SAP and its subsections. Every SAP consultant/developer should know these basics. Let’s start!

    1. SAP as a company/business:

  • Historically, it was established in 1972(one thousand nine hundred seventy-two) in Germany by 5 people from IBM.
  • It was originally called System Analysis Program Development, but was later shortened to SAP.
  • Technically, SAP is a system that facilitates the management of business processes of companies.
  • It is basically a program used in the field of ERP.

    2. ERP

  • ERP, which means Enterprise Resource Planning, can be defined as the general name given to systems that gather all the structure needed by an institution such as planning, production, operation... into a single body.
  • The most known and widespread program in the ERP world is SAP.
  • It is divided into modules to meet all needs. I can give examples such as SD(Sale and Distribution) module responsible for sales and distribution of companies and PP(Production Planning) module responsible for production and planning. We can reproduce it as ;

    3. ABAP and FIORI

  • ABAP (Advanced Business Application Programming) is an object-oriented, high-level software language developed by SAP for the development of SAP software.
  • With ABAP, the desired information is read from the database, and user interfaces that enable data exchange between SAP systems and fields required for adding records to the system can be written and reported.
  • FIORI is an ABAP is the user interface that enables screen outputs to be used more conveniently, easily and on different platforms.
  • It is developed with UI5 technology, which is a combination of User Interface and HTML5.

    4. The role of an SAP Consultant

  • The role of an SAP consultant is divided into 2 parts as Functional and Technical.
  • Functional consultants are responsible for driving business meetings, preparing business case documents and communicating client requirements to the technical consultants. They also configure the SAP system to run as per the client’s requirements. I just mentioned about modules, they working in these field and they are called SD, PP consultant.
  • Technical consultants are translate these documents(prepared by functional consultants) into technical documents and write code to implement their business needs.

    5. SAP S/4 HANA Cloud

  • S/4HANA is designed to solve complex problems and manage larger amounts of data. With SAP Fiori, it provides a better user experience and performs more complex operations on data.
  • While deploying the SAP S/4HANA suite, we'll come across S/4HANA Cloud.
  • SAP S/4HANA Cloud is a cloud ERP system that optimizes business processes with the help of intelligent automation and integrated technologies such as Artificial Intelligence (AI), Machine Learning and Business Analytics.
  • This enables us to achieve greater transparency, accessibility, agility, data accuracy and efficiency in our business.
  • Shortly, it is a cloud system where SAP S4/HANA developments are deployments.
